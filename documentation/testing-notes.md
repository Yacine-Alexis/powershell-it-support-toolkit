# Testing Notes

Use the following template for every script test record. This is the project evidence that AI-assisted code was tested rather than committed without verification.

```markdown
## Script

## Test environment

## Test cases

## Expected result

## Actual result

## Problems found

## Corrections made

## Remaining limitations
```

## Guidance

- Create one section per script and test date.
- Record both successful and failed test cases.
- Note environment details such as Windows version, PowerShell version, network conditions, and module availability.
- If a script depends on lab-only infrastructure, document that dependency explicitly.
- Keep example outputs in the `examples` folder when they help validate expected behavior.
